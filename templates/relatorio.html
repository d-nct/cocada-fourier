Séries de Fourier

Antes de nos aprofundar na matemática, vamos entender o que são as Séries de Fourier. A principal ideia por trás do método é decompor uma função com comportamento periódico em uma soma infinita de senos e cossenos, conhecidos como termos harmônicos.

Transformada de Fourier

A grande diferença entre as Séries de Fourier e as Transformadas de Fourier é que as transformadas não se limitam a funções com comportamento periódico!

O cientista da computação Kalid Azad faz uma analogia bem colocada em um post [2] sobre a Transformada de Fourier, comparando a técnica com uma vitamina, em que surgem as seguintes perguntas:

  • O que faz a Transformada de Fourier? Dada uma vitamina, encontra a receita.
  • Como? Passa a vitamina por diversos filtros para separar os ingredientes.
  • Por quê? Receitas são mais fáceis de analisar, comparar e modificar que a vitamina em si.
  • Como obtemos a vitamina de volta? Mistura tudo!

Agora que já temos uma abstração um pouco mais construída sobre a transformada, podemos partir para a matemática.

A Identidade de Euler pode ser compreendida geometricamente pela Figura 3. Fato é que um mesmo ponto em uma circunferência pode ser representado de mais de uma maneira:

  1. Como o vetor (cos⁡(x),sen(x))\big(\cos(x),\text{sen}(x)\big)(cos(x),sen(x))﻿; ou
  1. Como o número complexo eixe^{ix}eix﻿.

Desse modo, vale destacar que não é necessário utilizarmos números complexos — no lugar, podemos utilizar um par ordenado de números reais —, mas facilita a implementação do código. Isso porque fica mais claro trabalhar em cima de um vetor de números que de dois.

Visualizando

O simulador de ondas quadradas hospedado em /1 cobre basicamente toda a parte de animação. A magia em si dos desenhos é a Transformada de Fourier. A implementação mais básica simplesmente traduz o matematiquês para o código:

function transformadaDiscretaFourier(pontos) {
    let X = []; // vetor para armazenar os pontos da transformada
    let N = pontos.length; // número de pontos
    let real = 0; // variável auxiliar para somatório
    let imag = 0;
    let amplitude = 0;
    let fase = 0;
    let frequencia = 0;
    
    // itera pelos pontos do vetor da transformada
    for (let k = 0; k < N; k++) {
        real = 0;
        imag = 0;

        // itera pelos pontos da entrada
        for (let n = 0; n < N; n++) {
            xn = pontos[n];
            
            // calcula
            angulo = (2*PI * k * n) / N; // ganhei de graça da fórmula
            real += xn * cos(angulo);
            imag -= xn * sin(angulo);
        }

        real = real / N;
        imag = imag / N;

        // salvando metadados
        frequencia = k; // por definição
        amplitude = sqrt(real*real + imag*imag); // como é vetor, a amplitude é a norma
        fase = atan2(imag, real); // por definição

        X[k] = {real, imag, amplitude, fase, frequencia};
    }

    return X;
}

Note que na fórmula há apenas um somatório, mas no código há dois laços for. Isso ocorre porque estamos calculando o somatório para cada frequência kkk﻿. A complexidade dessa implementação é, portanto, da ordem de O(N2)O(N^2)O(N2)﻿.

Geralmente, a entrada do algoritmo da Transformada de Fourier seria um sinal de audio ou uma imagem. No nosso caso, a entrada será um vetor de pontos a ser recriado pelo método.

Desenhando uma Circunferência

Hospedado em /2, temos duas “máquinas de desenhar”:

  • A da esquerda recebeu um vetor de pontos (somente o eixo yyy﻿) gerado por sen(x)\text{sen}(x)sen(x)﻿, que foram processados utilizando transformadaDiscretaFourier.
    • O plot desses pontos de 000﻿ a 2π2\pi2π﻿ representa a “cabeça” dos epiciclos, representados em verde.
  • A de cima recebeu um vetor de pontos (somente o eixo yyy﻿) gerado por cos(x)\text{cos}(x)cos(x)﻿, que foram processados pelo mesmo método.
    • O plot desses pontos de 000﻿ a 2π2\pi2π﻿ representa a “cabeça” dos epiciclos, representados em vermelho.

As duas máquinas de desenhar se combinam para desenhar o ponto amarelo. Para tal, usa uma máquina de desenhar como cada entrada do ponto: (verde, vermelho) = amarelo.

Desenhando Qualquer Coisa

Partir de desenhar uma circunferencia para desenhar qualquer coisa não é um passo tão grande. Na verdade, é mais sobre a implementação da animação que sobre alterar o método da transformada.

No lugar da entrada ser o vetor de pontos gerado arbitrariamente por cos⁡\coscos﻿ ou sen\text{sen}sen﻿, a ideia é capturar o movimento horizontal e vertical do mouse. A partir daí, o programa é o mesmo: pré processamos esses pontos e plotamos com as máquinas de desenhar.

Essa versão está hospedada em /3, recomendo explorar a criatividade! Basta pressionar o botão esquerdo do mouse e desenhar sem tirar o lápis do papel.

Números Complexos

Essa versão está hospedada em /4. Para o usuário, o resultado é praticamente o mesmo: podemos desenhar livremente. A grande diferença é que no lugar de termos duas máquinas de desenhar que se combinam nos dois eixos, temos uma só que é capaz de desenhar nas duas dimensões.

Embora a teoria seja ligeiramente mais complexa pelo fato de usarmos números complexos no lugar de um par ordenado de números reais, a implementação é mais simples. Isso porque no lugar de tratarmos dois vetores de números separadamente para depois juntar, só precisamos lidar com um.

Para tal, foi necessário implementar uma classe Complexos para lidar com as operações entre números complexos e modificar a função transformadaDiscretaFourier para se integrar com a classe, mas o algoritmo é o mesmo.
